# Operation: Doctor — First Playable Boundary

## Purpose
This document defines the first playable boundary before implementation begins.

## First Playable Scope
The first playable must be small and controlled.

Included:
- One clinic scene
- Basic visitor queue
- Basic profile card
- Basic action selection
- Result-ready state
- Simple reward trigger

Excluded:
- Full content catalog
- Full staff system
- Full room system
- Full premium economy
- Store/release preparation
- Large production art pass

## Success Criteria
- The player can understand the basic loop.
- The scene is readable.
- The action flow is clear.
- The system is small enough to test.

## Risk
If the first playable scope grows, implementation will slow down and the project will lose control.
