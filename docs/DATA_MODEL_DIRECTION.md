# Operation: Doctor — Data Model Direction

## Purpose
This document defines the early data direction for Unity setup.

## Format Direction
Use JSON for early content data.

## Planned Data Files
```text
Assets/Data/visitors.json
Assets/Data/conditions.json
Assets/Data/checks.json
Assets/Data/actions.json
Assets/Data/staff.json
Assets/Data/rooms.json
```

## Data Rules
- Data must be readable.
- Data must be easy to edit.
- Data must not include real-world advice.
- Data should describe game states and abstract simulation values.
- Large content entry must wait until schema is stable.

## Stage 11 Boundary
Stage 11 defines data direction only.
No full data production starts yet.
